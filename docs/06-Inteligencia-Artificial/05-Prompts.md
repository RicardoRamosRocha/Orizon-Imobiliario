# Prompts

## Objetivo

Centralizar, padronizar e versionar todos os prompts utilizados pela plataforma, permitindo reutilização entre módulos e fácil manutenção.

## Estrutura

Prompt

- Nome
- Código
- Categoria
- Descrição
- Provider recomendado
- Modelo recomendado
- Prompt do Sistema
- Prompt do Usuário
- Variáveis
- Temperatura
- Máximo de Tokens
- Versão
- Status
- Data de Criação

## Categorias

- Atendimento
- Captação de Clientes
- Consultoria Jurídica
- Direito Imobiliário
- Compra e Venda
- Locação
- Marketing
- CRM
- Documentos
- Análise de Contratos
- Avaliação de Imóveis
- Chat Geral

## Variáveis

Exemplos:

- {{NomeCliente}}
- {{Cidade}}
- {{TipoImovel}}
- {{Valor}}
- {{Descricao}}
- {{Historico}}
- {{Documentos}}

## Funcionalidades

- Cadastro
- Edição
- Versionamento
- Duplicação
- Teste do Prompt
- Histórico de Alterações
- Importação e Exportação
- Biblioteca de Prompts

## Boas Práticas

- Utilizar prompts curtos e objetivos.
- Evitar informações duplicadas.
- Utilizar variáveis ao invés de texto fixo.
- Definir claramente o papel da IA.
- Limitar o escopo da resposta.
- Padronizar o formato da saída.

## Resultado Esperado

Uma biblioteca centralizada de prompts reutilizáveis, permitindo consistência nas respostas e evolução contínua da qualidade da IA.
